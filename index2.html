Introduction

A network is a system of connected devices that can exchange data and communicate. In practice, almost everything‚Äîpersonal computers, smartphones, servers, enterprise systems, and websites‚Äîparticipates in a network. The moment a device connects to the Internet, it becomes part of a vast, interoperable network.

Everyday view: Your laptop connects to a home Wi-Fi router (access point). When you type google.com in the browser, your computer sends a request to the router. The router forwards the request to the Internet, finds Google's servers, and returns the response (the web page) to your browser.

How a Web Request Works (High-Level)

  1. DNS Resolution: Convert a human-readable domain (e.g., google.com) into an IP address (e.g., 142.250.72.14).
  2. Routing: Packets are forwarded by routers across networks to reach the destination IP.
  3. Transport: TCP (reliable) or UDP (faster, no guarantee) delivers data to the correct application port.
  4. Application: Protocols like HTTP/HTTPS request the web page; the server responds with HTML/CSS/JS.
  5. Return Path: Responses follow routes back to your device; your browser renders the page.

Common Terms You Should Know

  • IP (Internet Protocol): The addressing system of the Internet‚Äîlike a street address for devices. Example: IPv4 192.168.1.10, IPv6 2001:0db8::1.
    • Public vs. Private IP: Private IPs (e.g., 192.168.x.x, 10.x.x.x) are used inside local networks; routers use NAT to map many private devices to a single public IP on the Internet.
  • DNS (Domain Name System): Translates human-readable domain names (e.g., naver.com) into IP addresses. It‚Äôs the Internet‚Äôs ‚Äúphone book.‚Äù
  • Client: A device or program that requests a service over the network (e.g., your web browser).
  • Server: A system that provides services or data to clients (e.g., a web server responding to HTTP requests).
  • LAN (Local Area Network): A small, local network‚Äîhome, school, or a single building.
  • MAN (Metropolitan Area Network): A city-scale network interconnecting multiple LANs across a metropolitan area.
  • WLAN (Wireless LAN): A LAN using Wi-Fi (wireless access points) instead of Ethernet cables.
  • WWAN (Wireless Wide Area Network): Wide-area wireless networks like 4G/5G cellular data.
  • WAN (Wide Area Network): A large network spanning regions or countries (the Internet is the ultimate WAN).

Core Concepts (Professional View)

  • Packets & Protocols: Data is split into packets with headers (addresses, ports, flags) and payloads. Protocols (IP, TCP, UDP, HTTP, TLS, etc.) define how packets are formed, routed, and interpreted.
  • OSI vs. TCP/IP Models:
    • OSI (7 layers): Physical ‚Üí Data Link ‚Üí Network ‚Üí Transport ‚Üí Session ‚Üí Presentation ‚Üí Application
    • TCP/IP (4‚Äì5 layers): Link ‚Üí Internet (IP) ‚Üí Transport (TCP/UDP) ‚Üí Application (HTTP, DNS, etc.)
    Understanding layers helps diagnose issues and design secure systems.
  • Ports & Services: Applications listen on ports (e.g., HTTP 80, HTTPS 443, DNS 53). Firewalls allow/deny traffic by port and protocol.
  • DHCP: Automatically assigns IP addresses, gateways, and DNS servers to devices on a network.
  • NAT: Network Address Translation lets many private devices share one public IP‚Äîcommon in homes and enterprises.
  • Switches vs. Routers:
    • Switch: Connects devices within a LAN (Layer 2), forwarding frames by MAC address.
    • Router: Connects different networks (Layer 3), forwarding packets by IP and routing tables.
  • HTTP ‚Üí HTTPS: HTTPS = HTTP over TLS (encryption + integrity + authentication). Always prefer HTTPS to prevent eavesdropping and tampering.

Security Perspective (for Aspiring Hackers)

  • Attack Surface: Open ports, outdated services, weak configurations (e.g., default credentials) enlarge risk.
  • Reconnaissance: DNS enumeration, port scanning, and service banner grabbing reveal targets and versions.
  • Segmentation: Separate critical systems (e.g., production DB) from user networks; use VLANs and firewalls.
  • Zero Trust & Least Privilege: Don‚Äôt trust by default; grant only the minimal access required.
  • Observability: Logs, flow records, and IDS/IPS (e.g., Suricata, Zeek) help detect anomalies and intrusions.

Conclusion

A network connects devices so they can communicate, but that simple idea spans many layers‚Äîaddressing (IP), name resolution (DNS), transport (TCP/UDP), and application protocols (HTTP/HTTPS). For cybersecurity, understanding these fundamentals is essential: it informs how attacks work and how to defend against them.

Start with the basics (IP, DNS, client/server, LAN/WAN) and gradually add advanced topics (routing, TLS, segmentation, monitoring). This foundation will support both ethical hacking and robust system design.
